# /phase-start

Start the next implementation phase.

1. Read `docs/progress.md` and find the next phase with status "not started".
2. Read the corresponding section in `docs/06-implementation-plan.md`.
3. Present to the user:
   - Phase number and title
   - Objective
   - Deliverables (summarised)
   - Acceptance criteria
   - Stop-if conditions
4. Update `docs/progress.md`: set the phase's status to "in progress" and fill in the "Started" date.
5. Ask the user to confirm before beginning work.
